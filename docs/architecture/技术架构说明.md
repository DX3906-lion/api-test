# 技术架构说明

## 1. 架构目标

本 Demo 用于验证自动化测试平台新脚本服务的核心链路：导入、草稿管理、脚本编排、变量中心、SQL 步骤、执行记录、正式脚本用例、复制/引用模式，以及 AI 助手接口预留。

平台功能应能独立运行，不依赖 AI 才能完成脚本编排；AI 作为增强能力，通过协议和 Patch Operations 接入。

## 2. 推荐技术栈

- 后端：Java 8 + Spring Boot 2.x
- 数据库：MySQL
- 前端 Demo：静态 HTML / Vue 2 均可
- 执行机：先用 mock executor 验证流程，再扩展真实 HTTP / SQL 执行
- AI：先实现 mock suggestions，再对接真实 AI 服务

## 3. 服务划分

```text
api-test
  ├── script-service        新脚本服务
  ├── executor-service      新执行机服务，Demo 阶段可 mock
  ├── ai-adapter            AI 接口适配层，Demo 阶段可 mock
  └── docs                  需求、原型、API、任务
```

第一阶段可以先做单体 Spring Boot Demo，按包结构隔离模块；后续再拆分微服务。

## 4. 逻辑模块

### 4.1 导入与草稿模块

负责导入 HAR 文件、校验格式、生成脚本草稿、生成接口步骤、草稿列表查询、草稿详情查询。

### 4.2 编排模块

负责脚本步骤的新增、编辑、删除、复制、拖拽排序、启用禁用，以及 API_STEP、SQL_STEP、IF_ELSE、FOR_LOOP、WAIT、COMPONENT 等节点模型。

### 4.3 报文与字段配置模块

负责 `rawBody + fieldConfigs + treeCache`。

- rawBody：干净可执行报文。
- fieldConfigs：参数化、备注、字段配置、断言/提取关联。
- treeCache：展示缓存。

### 4.4 变量中心模块

负责脚本变量、变量来源、变量作用域、变量使用位置、变量校验。

### 4.5 SQL 步骤模块

负责 SQL 步骤配置、SQL 环境信息填写、保存到环境配置、SQL 调试结果记录。

### 4.6 执行记录模块

负责单步调试、全流程调试、执行记录列表、执行记录详情、逻辑步骤树、报文信息、执行日志、变量情况。

### 4.7 版本与用例模块

负责草稿保存为脚本版本和用例版本，正式脚本用例列表，编辑生成草稿，复制正式用例且强制修改脚本名称。

### 4.8 复用模块

负责组件、单接口脚本、可复用步骤的复制模式和引用模式。引用模式支持固定版本和最新版本，支持进入被引用对象编排页面。

### 4.9 AI 接口预留模块

负责 AI 输入协议、输出协议、Patch Operations、预览 Diff、应用 Patch、草稿校验。AI 不直接改库，平台负责落草稿。

## 6. 调用链路

### 6.1 导入链路

```text
上传 HAR
  ↓
校验 HAR 格式和 log.entries
  ↓
保存原始文件元信息
  ↓
解析 entries
  ↓
生成 ScriptDraft
  ↓
生成 FlowSteps / PayloadContent / FieldConfigs
  ↓
进入编排页面
```

### 6.2 编排保存链路

```text
用户编辑步骤
  ↓
保存步骤详情
  ↓
保存 rawBody / fieldConfigs / variables / extractors / assertions
  ↓
validateDraft
  ↓
保存草稿
```

### 6.3 草稿发布链路

```text
当前草稿
  ↓
版本对比
  ↓
用户确认
  ↓
生成 ScriptVersion
  ↓
生成 CaseVersion
  ↓
进入正式脚本用例列表
```

### 6.4 AI 采纳链路

```text
用户输入自然语言
  ↓
Context Builder 按需组装上下文
  ↓
AI 返回 suggestions + operations
  ↓
previewPatch
  ↓
用户采纳
  ↓
applyDraftPatch
  ↓
validateDraft
  ↓
保存草稿或调试
```

## 7. 执行机设计建议

Demo 阶段执行机可以 mock 返回结果，但数据结构应按真实执行记录设计。

执行机应支持：

- API_STEP HTTP 请求。
- SQL_STEP 调试和执行。
- 变量上下文渲染。
- 平台函数渲染。
- 断言执行。
- 变量提取。
- 执行日志。

## 8. AI 接口预留边界

平台提供上下文查询、Patch 预览、Patch 应用、草稿校验。AI 负责建议生成，不负责持久化。

关键接口：

- `/api/ai/assist`
- `/api/ai/patch/preview`
- `/api/ai/patch/apply`
- `/api/scripts/{draftId}/validate`

## 9. Codex 开发建议

先做平台主链路，不要先做真实 AI：

1. 项目骨架
2. 数据模型
3. 导入与草稿
4. 编排基础接口
5. 变量中心与 fieldConfigs
6. 提取器与断言
7. SQL 步骤
8. 执行记录
9. 正式脚本用例与复制
10. 复制/引用模式
11. AI 接口预留
