# 新脚本服务需求说明书 v12

## 1. 总体定位

本项目建设自动化测试平台的新脚本服务 Demo。平台功能主线可以独立推进，并预留 AI 助手接口。新服务与旧服务并行，执行计划通过 `engineVersion` 路由到旧执行机或新执行机。

新服务覆盖：

- 导入 / 草稿管理，当前导入功能支持 HAR 与 Postman Collection 格式。
- 脚本草稿与编排页面。
- API_STEP、SQL_STEP、IF_ELSE、FOR_LOOP、WAIT、COMPONENT。
- `rawBody + fieldConfigs + treeCache` 报文模型。
- 变量中心、字段配置、变量提取、断言配置。
- SQL 步骤编辑与调试。
- 组件 / 单接口脚本 / 可复用步骤的复制模式与引用模式。
- 执行记录内嵌到脚本内部。
- 草稿保存为脚本版本与用例版本。
- 正式脚本用例复制时必须修改脚本名称。
- AI 助手接口预留。

## 2. 系统工作空间

保留右上角系统切换。用户可以拥有多个系统权限，切换系统后进入对应工作空间。

系统切换影响：导入草稿、脚本草稿、正式脚本用例、交易信息匹配、执行记录、当前系统默认筛选范围。

系统切换不强制影响：用户全局变量、用户环境配置、用户组件库、用户外部数据集。以上资源为用户维度，可配置适用系统。

## 3. 页面结构

主要页面：

1. 导入 / 草稿管理页面
2. 脚本编排页面
3. 正式脚本用例列表页面
4. 用户全局变量页面
5. 用户环境配置页面
6. 用户组件库页面
7. 用户外部数据集页面

## 4. 导入 / 草稿管理

导入页面上方提供导入按钮、新建空白脚本按钮、搜索、筛选、刷新、批量删除。页面下方展示草稿列表。

当前第一阶段导入功能支持 HAR 与 Postman Collection 格式。导入弹窗必填脚本草稿名称和导入文件，自动带出当前系统、当前用户、导入时间。

导入校验：

- 脚本草稿名称不能为空。
- 导入文件不能为空。
- 当前支持 HAR 与 Postman Collection 格式。
- HAR 文件必须包含 `log.entries`；Postman Collection 必须包含 `item`。
- 文件大小不能超过限制。
- 同系统下草稿名称重复时提示。

导入成功后：保存原始导入文件，解析 HAR log.entries 或 Postman item，过滤静态资源、OPTIONS、埋点、第三方请求，生成脚本草稿和接口步骤，步骤使用 `rawBody + fieldConfigs + treeCache`，并自动进入编排页面。

## 5. 脚本编排页面

编排页面由顶部操作区、左侧步骤编排区、右侧步骤详情区和底部调试结果抽屉组成。

顶部操作包括：返回草稿列表、保存草稿、单步调试、全流程调试、执行记录、变量中心、版本对比、保存为脚本用例、AI 助手。

左右区域支持缩放：默认 50/50、编排优先、详情优先。

## 6. 编排步骤类型

- API_STEP：接口步骤
- SQL_STEP：SQL 步骤
- IF_ELSE：条件判断
- FOR_LOOP：循环
- WAIT：等待
- COMPONENT：组件
- PRE_PROCESSOR：前置处理
- POST_PROCESSOR：后置处理

左侧步骤支持拖拽排序、复制、删除、启用/禁用、批量命名、单条命名、生成组件、添加接口步骤、添加 SQL 步骤、引入单接口脚本、导入 cURL、添加 IF-ELSE、添加 FOR、添加 WAIT。

## 7. 请求与报文模型

请求配置展示完整 URL，并支持 baseUrl、path、query、header、cookie、body 字段参数化。

支持请求格式：JSON、XML、multipart/form-data、x-www-form-urlencoded、raw text、binary/file、custom、BUMS 协议报文。

报文模型：

- `rawBody`：报文事实源，只保存干净可执行报文。
- `fieldConfigs`：保存字段参数化、备注、断言、提取等配置。
- `treeCache`：展示缓存，执行逻辑不能依赖。

字段备注存储到 `fieldConfigs.remark`，展示时可渲染为注释，实际发送不带备注。

## 8. 字段配置与参数化

字段配置字段包括：字段路径、配置类型、变量名、变量中文名、变量英文名、数据类型、默认值、备注、状态。

配置类型包括：参数化、常量字段、函数生成、忽略字段。

用户可以直接使用 `${变量英文名}`，也可以选中原始报文字段值后设为变量。前端通过编辑器选区、格式解析器 AST 和字段路径定位识别字段，确认后替换 rawBody、写入 fieldConfigs，并注册到变量中心。

平台函数示例：

- `fn.uuid()`
- `fn.now()`
- `fn.timestamp()`
- `fn.randomInt(1,100)`
- `fn.randomString(8)`
- `fn.dateAdd(fn.now(), 1, 'day')`
- `fn.md5(customerId)`
- `fn.base64(payload)`

变量使用时可做简单函数处理，例如 `${customerId | trim}`、`${amount | numberFormat('0.00')}`、`${applyDate | dateFormat('yyyy-MM-dd')}`。

## 9. 变量中心

变量中心是统一管理入口，但不是唯一创建入口。变量可以来自变量中心手工新增、请求体字段参数化、URL/Header/Query 参数化、响应提取器、SQL 结果提取、组件输出映射、外部数据字段引用、AI 建议采纳、直接文本 `${xxx}`。

变量属性包括：变量中文名、变量英文名、数据类型、默认值、变量来源、作用域、描述、状态、使用位置、产生步骤。

数据类型：STRING、NUMBER、BOOLEAN、OBJECT、ARRAY、DATE、FILE。

变量来源：MANUAL、GLOBAL_REF、ENV_REF、EXTRACTOR、SQL_RESULT、COMPONENT_OUTPUT、DATASET_FIELD、AI_SUGGESTED、FUNCTION、SYSTEM_RUNTIME。

作用域：SCRIPT、STEP、LOOP、COMPONENT、RUNTIME。

保存草稿、调试、发布前校验变量是否定义、英文名是否重复、是否使用前产生、是否存在循环依赖、表达式是否有效、数据类型是否匹配。

## 10. 变量提取与断言

变量提取支持来源：响应体、响应头、Cookie、SQL 结果、组件输出、正则、自定义路径。

表达式类型：JSONPath、XPath、HEADER、COOKIE、REGEX、SQL_RESULT、CUSTOM_PATH。

断言支持：状态码断言、JSONPath 断言、XPath 断言、Header 断言、Cookie 断言、响应体包含/不包含、正则断言、响应时间断言、SQL 结果断言、字段存在/不存在。

响应信息在步骤详情底部展示，使用与顶部选项卡一致的风格，包括基本信息、响应头、响应体、断言与变量。

## 11. SQL 步骤

SQL 步骤支持 MySQL、Oracle、OceanBase-Oracle。

配置项：步骤名称、数据库类型、数据源/连接配置、SQL 类型、SQL 内容、超时时间、失败策略、变量提取规则、断言规则。

SQL 步骤编辑时支持调试，展示 SQL 语句、数据库类型、执行耗时、影响行数、查询结果预览、错误信息、变量提取结果、断言结果。

用户可以在 SQL 步骤页面直接填写 SQL 环境信息，包括数据库类型、JDBC URL、数据库名/Schema、用户名、连接参数、超时时间。用户可选择仅本步骤使用，或保存到我的环境配置。

## 12. 复制模式与引用模式

组件、单接口脚本、可复用脚本步骤支持复制模式和引用模式。

复制模式：引入时复制为当前脚本草稿中的独立步骤和独立配置，后续互不影响。

引用模式：当前脚本保存引用对象类型、对象 ID、版本 ID、固定版本/最新版本、输入映射、输出映射、覆盖配置。引用步骤可点击进入被引用对象的编排页面；被引用对象生成新版本后，当前脚本可选择是否升级引用。

## 13. 执行记录

执行记录放在脚本内部。点击执行记录按钮弹出执行记录列表。选中一条记录后打开详情弹窗，左侧展示逻辑步骤树，右侧展示基本信息、报文信息、执行日志、变量情况。

## 14. 保存与版本管理

保存草稿只更新 ScriptDraft、FlowSteps、PayloadContent、Variables、Assertions、Extractors、FieldConfigs，不生成正式版本。

保存为脚本用例时，同步生成 ScriptVersion 和 CaseVersion。已保存版本不可直接修改。

版本对比支持已保存脚本版本与当前草稿对比，内容包括步骤、URL、Header、Query、Body、fieldConfigs、变量、断言、提取器、SQL、组件、交易信息关联、失败策略。

## 15. 正式脚本用例复制

正式脚本用例支持复制。复制时必须填写新的脚本名称。新名称不能为空，且不能与当前系统下已有脚本重复。第一阶段建议复制后生成新草稿，用户确认后再保存为正式脚本用例。

## 16. AI 助手预留

AI 助手只作用于当前脚本草稿，不直接修改正式版本或交易信息正式版本。AI 通过输入协议获取上下文，输出结构化 suggestions 和 Patch Operations。用户采纳后由平台执行 patch，并在保存或调试前校验。

## 17. 主流程

```text
用户选择系统
  ↓
进入导入 / 草稿管理页面
  ↓
导入 HAR / Postman Collection 或新建空白脚本
  ↓
生成脚本草稿
  ↓
进入编排页面
  ↓
添加 API / SQL / IF / FOR / WAIT / COMPONENT 步骤
  ↓
配置报文、字段配置、备注、变量、断言、SQL、组件输入输出、外部数据
  ↓
选择复制模式或引用模式引入组件 / 单接口脚本
  ↓
单步调试 / 全流程调试
  ↓
查看脚本内部执行记录
  ↓
版本对比
  ↓
同步生成脚本版本和用例版本
  ↓
进入正式脚本用例列表
  ↓
复制正式用例时必须修改脚本名称
  ↓
加入执行计划执行
```
